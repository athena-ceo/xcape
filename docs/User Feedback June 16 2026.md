## Harley Comments on this


* Maybe there is a smooth way to go from an ultra-simple view to a more sophisticated one?   Right now there is some progressive disclosure but maybe there is not enough difference between casual and serious use?
* Sorting the list of criteria by importance to that person and relevance to the persona seems like an easy good idea.   Maybe a simple “sort by…” affordance in the table?
* In any case the key persona criteria should be filters, and it should be easier to set up filters (even in the table, maybe have sort / filter headers on the criteria like a spreadsheet).   Also maybe countries should be on the y axis and criteria on the x axis?   To be mocked up to compare.
* The criteria value links should clearly be underlined like browser links and maybe have a tooltip or be highlighted in a meaningful way when moused over (not sure how this works on mobile which is key)
* It seems we need more detailed and relevant criteria for fiscal protection as a persona with more specific information about real estate, assets, banking, wealth taxation, and so on.
* For community protection, right now it just has a generic “community protection” criterion and then the specific community criteria are added (as a special case?) under “Your criteria” - but no, these are part of the security / safety criteria and they are essential filters.  For community safety, there are other sub criteria that would be interesting:  trends in anti-semitic or islamophobic or homophobic acts as reported, links and information from relevant groups (e.g. ADL for Jews).  People who are worried about politics (takeover of France by the RN or LFI) will have similar detailed concerns and want to look at countries that are broadly sympathetic.
* Cleary need to minimise the number of buttons and affordances in the default display and make the ones visible super easy to identify.
